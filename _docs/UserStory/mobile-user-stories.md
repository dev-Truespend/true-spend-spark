# Mobile User Stories

This document follows the current mobile mockup in `_docs/MobileApp-Mockup/index.html`.
Stories are grouped only by delivery phase, not by Basic or Pro tier.

## Phase 1 User Stories

Phase 1 covers screens `1.1` through `8.5`: auth, onboarding, in-store recommendations, cards, the Insights tab (manual and Plaid-imported transactions), notifications, profile, billing, privacy, and sync.

Scope notes for Phase 1:

- Plaid is used for card metadata and linked transaction import.
- The Phase 1 bottom tab bar is: Home, Cards, Insights, Profile. The Search tab (online purchase helper) ships in Phase 2.
- Receipt scanning, budgets, and geofence management (list/pause/edit/map UI in `P2.7`, `P2.8`) are Phase 2.
- The geofence-triggered lock-screen card nudge (`P2.7b`) is **Phase 1**, powered by Foursquare background detection — but the user-facing geofence management UI stays Phase 2.
- Billing is processed by Stripe via hosted checkout and customer portal; checkout should surface Apple Pay on iOS and Google Pay on Android when available. AI insights are generated by Azure OpenAI. Background geofence detection is powered by Foursquare.
